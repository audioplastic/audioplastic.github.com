Compile Mex Files With XCode 5 on OSX Mavericks Using Older Versions of Matlab

Feb 3rd, 2014 | Comments

So, you’ve upgraded OSX to the latest version and also upgraded XCode to 5.X. You’ll find that the Matlab mex command now fails to build your Matlab executables files as compiler related stuff has moved and version numbers have changed. Thankfully, the fix is relatively simple. Just locate mexopts.sh, which can be found in ~/your_old_matlab_version/mexopts.sh, and open it with your favorite editor.
